# Week 05 - Transformer

## 개요

이번 위클리 챌린지는 이미지 분류 모델을 직접 불러오고, 전이 학습과 모델 비교, 하이퍼파라미터 튜닝 흐름을 기초적으로 실습하는 과제입니다.

주요 작업 파일은 `week05-transformer.ipynb`입니다. 노트북에서는 EuroSAT 이미지 데이터셋을 사용해 ResNet과 VGG16 모델을 다루고, 별도의 가상 데이터셋으로 GridSearch와 RandomSearch를 실습했습니다.

---

## 섹션별 구현 내용

### 1. ResNet 모델과 데이터셋 준비

먼저 `resaro/eurosat` 데이터셋을 불러오고, 샘플 이미지를 출력해서 이미지와 라벨이 어떤 형태로 들어 있는지 확인했습니다.

그 다음 ResNet50 모델을 불러왔습니다. 이 단계에서는 모델 구조를 준비하는 것이 목적이었기 때문에, 사전 학습 가중치를 사용하지 않고 `weights=None`으로 불러왔습니다.

데이터셋은 이후 VGG16 전이 학습과 ResNet / VGG16 비교에서도 함께 사용할 수 있도록 `split_dataset`으로 정리했습니다.

- 기존 `train` 데이터의 20%를 `validation` 데이터로 분리했습니다.
- 기존 `test` 데이터는 최종 평가용으로 그대로 유지했습니다.
- `DatasetDict`를 사용해 `train`, `validation`, `test` 구조로 다시 묶었습니다.

**데이터 분할에서 배운 점**

처음 데이터셋을 확인했을 때는 train 1000개, test 1000개로 이미 나뉘어 있었습니다. 처음에는 데이터를 나눈다고 하면 6:2:2처럼 전체 데이터를 train / validation / test로 다시 나누는 방식만 떠올렸습니다.

하지만 이 경우에는 test 데이터가 이미 따로 제공되어 있었기 때문에, test는 학습이나 모델 선택 과정에 사용하지 않고 마지막 평가용으로 남겨두는 것이 더 적절했습니다. 대신 train 데이터의 일부를 validation으로 분리하면, test 데이터를 건드리지 않으면서 학습 중 모델 상태를 확인할 수 있습니다.

기존에 알고 있던 6:2:2 방식과 달라서 신기했고, 데이터 분할은 정해진 비율을 무조건 적용하기보다 데이터셋이 처음부터 어떻게 제공되었는지에 맞춰 판단해야 한다는 점을 알게 되었습니다.

### 2. VGG16 전이 학습

- 사전 훈련된 VGG16 모델이 기대하는 입력 형태에 맞게 이미지를 전처리했습니다.
- `DataLoader`를 만들어 train / validation / test 데이터를 batch 단위로 사용할 수 있게 했습니다.
- VGG16의 특징 추출 부분은 고정하고, 현재 데이터셋의 클래스 개수에 맞는 분류기 부분을 새로 정의했습니다.
- `CrossEntropyLoss`와 `Adam` optimizer를 사용해 기본적인 학습 루프를 작성했습니다.
- 학습 후 test 데이터로 정확도를 확인했습니다.

### 3. ResNet과 VGG16 성능 비교

- 2번에서 사용한 것과 동일한 데이터셋과 DataLoader를 사용했습니다.
- 사전 훈련된 ResNet50 모델을 불러와 마지막 분류 layer만 현재 클래스 개수에 맞게 바꿨습니다.
- VGG16과 ResNet50을 같은 조건에서 학습한 뒤 test loss와 test accuracy를 비교했습니다.

### 4. GridSearch와 RandomSearch

- `make_classification`으로 가상 분류 데이터셋을 만들었습니다.
- `RandomForestClassifier`를 기준 모델로 사용했습니다.
- `GridSearchCV`로 정해진 하이퍼파라미터 조합을 모두 탐색했습니다.
- `RandomizedSearchCV`로 일부 조합만 무작위로 탐색했습니다.
- 두 방식이 찾은 best parameter와 test accuracy를 비교했습니다.
