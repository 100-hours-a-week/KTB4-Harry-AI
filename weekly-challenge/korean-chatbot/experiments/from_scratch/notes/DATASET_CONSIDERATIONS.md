# 데이터셋 선택 시 고려할 점

이 문서는 나중에 실제 한국어 데이터셋을 고를 때 다시 보기 위한 메모다.
초기 구현은 작은 직접 작성 seed corpus로 시작하지만, 이후 공개 데이터셋을
검토할 때 아래 질문들을 사용한다.

## 기본 기준

- 우선은 한국어를 기준으로 한다.
- 영어도 함께 처리할 수 있으면 좋지만, 첫 목표는 한국어 흐름 이해다.
- 공개 포트폴리오에 사용해도 안전한 라이선스여야 한다.
- 출처와 사용 조건을 README나 ADR에 남길 수 있어야 한다.
- 처음부터 너무 큰 데이터셋을 전부 쓰기보다 일부 샘플로 작은 실험을 할 수 있어야 한다.
- 노이즈가 너무 많아 전처리만 하다가 모델 학습으로 넘어가지 못하는 데이터는 피한다.
- 다만 적당한 노이즈는 전처리 연습에 도움이 될 수 있다.

## 데이터 출처 후보

- 직접 작성한 작은 seed corpus
- 공개 라이선스가 명확한 한국어 문장 데이터
- 위키피디아 문장 또는 문서 일부
- 공개 대화 데이터
- 뉴스, 리뷰, Q&A 데이터

실제 사용 전에는 반드시 라이선스와 사용 조건을 확인한다.

## 한국어 전처리에서 볼 점

한국어는 조사, 어미, 형태소가 문장 의미와 문법에 큰 영향을 준다. 다만 첫
구현에서는 character-level 토큰화로 시작하므로 형태소 분석을 바로 도입하지
않는다.

나중에 개선할 때 검토할 수 있는 주제:

- 조사와 어미를 어떻게 다룰 것인가?
- 띄어쓰기 오류를 얼마나 허용할 것인가?
- 형태소 분석기를 사용할 것인가?
- subword tokenizer를 학습할 것인가?
- 문장부호와 숫자를 어떻게 처리할 것인가?

## 영어까지 확장할 때 볼 점

영어 데이터까지 다룰 경우 다음 전처리 이슈가 생길 수 있다.

- 대소문자 통일 여부
- 축약형 처리
- 불용어 제거 여부
- 문장부호 처리
- 한국어와 영어를 같은 tokenizer로 다룰지 여부

다만 언어 모델 학습에서는 불용어가 항상 제거 대상은 아니다. 문장을 생성하는
모델은 조사, 관사, 접속사 같은 작은 단어도 문장 흐름에 필요할 수 있다.

## 말투와 corpus의 관계

모델은 학습 corpus의 말투를 따라갈 가능성이 높다.

- 뉴스 데이터로 학습하면 문어체와 기사체가 강해질 수 있다.
- 위키 데이터로 학습하면 설명문 스타일이 강해질 수 있다.
- 대화 데이터로 학습하면 구어체와 짧은 응답이 늘어날 수 있다.
- 직접 작성한 seed corpus로 학습하면 작성자의 문장 습관에 치우칠 수 있다.

이 말투의 차이는 넓은 의미에서 persona와 연결될 수 있다. 다만 persona는
단순한 말투뿐 아니라 역할, 지식 범위, 응답 태도까지 포함하는 더 넓은 개념이다.

## 이후 전략 아이디어

아직 확정한 전략은 아니지만, 다음 방향을 검토할 수 있다.

1. 작은 seed corpus로 전체 학습 파이프라인을 이해한다.
2. 문어체 또는 위키체 데이터로 기본 문장 생성 능력을 넓힌다.
3. 대화체 데이터로 응답 스타일을 조정한다.
4. 각 단계에서 생성 결과와 한계를 기록한다.

이 전략은 pre-training과 fine-tuning의 축소판처럼 볼 수 있지만, 초반에는
용어보다 실제 실험과 관찰을 우선한다.
