# 티칭 페어 프로그래밍 계획

## 전략

첫 번째 소주제의 목표는 "성능 좋은 챗봇 만들기"가 아니다. 목표는
"내가 설명할 수 있는 챗봇 모델 만들기"다.

작업은 작고 관찰 가능한 단계로 진행한다. 각 파일은 하나의 실용적인
모듈을 나타내고, 한 번의 학습 세션에서는 그 모듈 안에 작은 함수, 클래스,
검증 중 하나만 추가한다. 헷갈리는 순간에는 코드를 더 추가하지 말고 값,
텐서 shape, 예시를 먼저 확인한다.

## 세션 진행 방식

매 학습 세션은 다음 흐름을 따른다.

1. 이전 기록을 읽는다.
2. 아주 작은 작업 하나를 고른다.
3. 예상 입력과 예상 출력을 한국어로 적어본다.
4. 필요한 최소 코드만 구현한다.
5. 해당 파일을 직접 실행한다.
6. 실행 결과를 노트에 설명한다.
7. 다음 세션에서 개선할 아이디어 하나를 남긴다.

## 티칭 방식

어시스턴트는 코드를 작성하면서 다음 방식으로 설명한다.

- 새로운 Python 또는 PyTorch 문법은 사용하기 전에 먼저 설명한다.
- 중간값을 숨기지 않고 직접 보여준다.
- 영리한 추상화보다 명확한 이름을 우선한다.
- 아직 학습자가 확인할 수 없는 기능은 추가하지 않는다.
- 전체 앱을 실행하지 않아도 각 단계를 따로 실행할 수 있게 유지한다.

## 포트폴리오 스토리

포트폴리오에서는 결과물만이 아니라 과정을 강조한다.

- 작은 from-scratch 한국어 언어 모델에서 시작했다.
- 토큰화, 텐서, causal language modeling, 학습, 생성을 단계적으로 익혔다.
- 부족한 점을 관찰하고 기록했다.
- 매주 하나씩 개선을 시도했다.
- 이후 RAG, LangChain, graph database, agent, MCP 같은 기술이 실제 한계를
  해결하는지 검토했다.

## 파일 구조

너무 많은 파일 대신 여섯 개의 파일로 시작한다.

1. `src/step01_tokenizer.py`
2. `src/step02_dataset.py`
3. `src/step03_tiny_model.py`
4. `src/step04_train_loop.py`
5. `src/step05_generate.py`
6. `src/step06_fastapi.py`

이 구조는 의도적으로 튜토리얼과 실제 프로젝트의 중간 지점을 목표로 한다.
교재식 노트북처럼 너무 잘게 쪼개지는 것은 피하되, 완성형 패키지보다
읽기 쉽도록 유지한다.

## 첫 번째 추천 마일스톤

FastAPI를 다루기 전에, 작은 말뭉치에 overfit할 수 있고 짧은 문장을 생성할
수 있는 character-level 한국어 미니 언어 모델을 만든다.

성공 기준은 다음과 같다.

- 토큰 id가 무엇인지 설명할 수 있다.
- 입력 텐서와 정답 텐서를 설명할 수 있다.
- 모델이 왜 다음 토큰을 예측하는지 설명할 수 있다.
- auto-regressive 생성 루프를 설명할 수 있다.
- 생성 문장이 어색해도, 전체 파이프라인은 이해할 수 있다.
