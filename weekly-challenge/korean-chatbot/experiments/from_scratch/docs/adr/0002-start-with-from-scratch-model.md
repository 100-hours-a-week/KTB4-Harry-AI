# ADR 0002. 학습용 구현은 from-scratch 모델로 시작한다

- 상태: Accepted
- 날짜: 2026-06-18
- 관련 단계: `src/step03_tiny_model.py`

## 배경

이번 과제의 주제는 사용자가 문장을 입력하면 다음 단어 또는 토큰을 생성하고,
그 과정을 반복해 완전한 문장을 생성한 뒤 FastAPI로 사용할 수 있게 만드는
것이다.

외부 LLM API나 pretrained 챗봇 엔진을 사용하면 빠르게 그럴듯한 결과를 만들
수 있지만, 이 프로젝트의 핵심 목표는 높은 성능보다 내부 흐름을 이해하는
것이다.

## 문제

챗봇 결과만 보면 외부 API 기반 프로젝트와 from-scratch 모델 프로젝트가
비슷해 보일 수 있다. 하지만 포트폴리오 관점에서는 "무엇을 직접 이해하고
구현했는가"가 중요하다.

## 고려한 선택지

1. 외부 hosted LLM API를 사용한다.
2. Hugging Face pretrained 모델을 가져와 fine-tuning 또는 inference만 한다.
3. 아주 작은 PyTorch Transformer 언어 모델을 직접 구현한다.

## 결정

첫 학습 트랙은 PyTorch 기반의 작은 from-scratch Transformer 언어 모델로
시작한다.

## 결정 이유

- 토큰화, 텐서, 모델 forward, loss, 학습 루프, 생성 과정을 직접 설명할 수 있다.
- 성능은 낮아도 학습 과정 자체가 포트폴리오 스토리가 된다.
- 이후 pretrained 모델이나 RAG를 붙이더라도, 무엇이 해결되는지 비교할 수 있다.
- 강한 모델을 사용하는 것보다 작은 모델의 한계를 직접 보는 것이 학습에 유리하다.

## 감수하는 단점

- 초기 생성 품질은 낮을 가능성이 높다.
- 좋은 데이터와 긴 학습 없이 실사용 챗봇처럼 동작하기 어렵다.
- 직접 구현해야 하므로 초반 속도는 느리다.

## 나중에 다시 볼 조건

from-scratch 모델의 한계를 충분히 관찰한 뒤, 성능 개선이 목적이 되면
pretrained 모델, tokenizer 개선, RAG, LangChain, LangGraph 등을 별도 ADR로
검토한다.

## 결과 기록

첫 구현은 작은 말뭉치에 overfit할 수 있는 수준을 목표로 한다. 좋은 챗봇이
아니라 "언어 모델 학습과 생성 파이프라인을 설명할 수 있는 상태"가 1차
성공 기준이다.
