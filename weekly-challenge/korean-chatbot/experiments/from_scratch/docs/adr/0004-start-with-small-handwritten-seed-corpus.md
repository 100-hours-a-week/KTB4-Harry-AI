# ADR 0004. 첫 학습 데이터는 작은 직접 작성 seed corpus로 시작한다

- 상태: Accepted
- 날짜: 2026-06-18
- 관련 단계: `src/step01_tokenizer.py`, `src/step02_dataset.py`

## 배경

언어 모델을 학습하려면 corpus가 필요하다. corpus는 모델이 학습할 원천
텍스트 말뭉치를 뜻한다.

이 프로젝트의 첫 목표는 좋은 성능의 챗봇을 바로 만드는 것이 아니라,
텍스트가 토큰이 되고, 토큰 id가 dataset이 되고, dataset이 모델 학습과
생성으로 이어지는 전체 흐름을 이해하는 것이다.

## 문제

처음부터 외부 한국어 데이터셋을 사용하면 데이터 수집, 라이선스 확인,
파일 포맷 해석, 정제, 필터링, 중복 제거, 문장 분리 같은 문제가 한꺼번에
등장할 수 있다.

이 과정은 중요하지만, 첫 단계에서 너무 많은 복잡도를 만들면 tokenizer,
dataset, model, train loop의 기본 흐름을 이해하기 전에 전처리 문제에
막힐 수 있다.

## 고려한 선택지

1. 공개 한국어 데이터셋을 바로 찾아서 사용한다.
2. 위키피디아 같은 공개 문서에서 일부 문장을 가져온다.
3. 직접 작성한 작은 한국어 seed corpus로 먼저 파이프라인을 검증한다.
4. 외부 LLM이나 다른 도구로 학습 문장을 생성한다.

## 결정

첫 구현은 직접 작성한 작은 한국어 seed corpus로 시작한다.

## 결정 이유

- 데이터 품질과 문장 형태를 직접 통제할 수 있다.
- 라이선스와 출처 문제를 초반에 단순화할 수 있다.
- corpus가 작으면 tokenization, dataset 생성, 학습, 생성 결과를 빠르게 확인할 수 있다.
- 복잡한 전처리보다 전체 파이프라인 이해에 집중할 수 있다.
- 나중에 외부 데이터셋을 붙였을 때 무엇이 어려워졌는지 비교하기 쉽다.

## 감수하는 단점

- 문장 수가 적어 일반화 성능은 거의 기대하기 어렵다.
- 직접 작성한 말투와 주제에 모델 출력이 강하게 치우칠 수 있다.
- 실제 한국어 데이터셋에서 발생하는 노이즈와 전처리 문제를 충분히 경험하기 어렵다.

## 나중에 다시 볼 조건

작은 seed corpus로 tokenizer, dataset, tiny model, train loop, generation을
한 번 연결한 뒤에는 공개 한국어 데이터셋을 검토한다.

그때는 다음 기준을 확인한다.

- 공개 포트폴리오에 사용 가능한 라이선스인가?
- 한국어 문장 품질이 적절한가?
- 문어체, 구어체, 뉴스체, 위키체 등 말투가 무엇인가?
- 너무 많은 노이즈로 인해 전처리 학습보다 데이터 청소에만 시간이 쏠리지 않는가?
- 일부 샘플만 잘라서 작은 실험에 사용할 수 있는가?

## 결과 기록

다음 단계에서는 `korean-chatbot/data/seed_corpus.txt`를 만들고, 짧은 한국어
문장으로 tokenizer와 dataset 흐름을 먼저 검증한다. 이 corpus는 최종
데이터셋이 아니라 학습 파이프라인 점검용 데이터다.
