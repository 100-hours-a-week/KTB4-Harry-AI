# Architecture Decision Records

이 폴더는 한국어 챗봇 학습 프로젝트의 주요 의사결정을 기록한다.

ADR은 "왜 이 방식을 선택했는가"를 남기기 위한 문서다. 코드와 모델 성능은
나중에 바뀔 수 있지만, 당시의 문제 인식과 선택 이유는 시간이 지나면 쉽게
잊힌다. 이 프로젝트에서는 그 과정을 포트폴리오와 면접 답변의 근거로 남긴다.

## 언제 ADR을 작성하는가

모든 생각을 ADR로 만들지는 않는다. 아래 질문에 해당하면 ADR을 작성한다.

- 나중에 면접에서 "왜 이 방식을 선택했나요?"라고 물을 수 있는가?
- 다른 선택지도 있었지만 하나를 골랐는가?
- 이후 코드 구조, 데이터 처리, 모델 성능, 실험 방향에 영향을 주는가?
- 몇 주 뒤에 다시 보면 선택 이유를 잊을 가능성이 있는가?

## 작성 규칙

- 파일명은 `0001-short-title.md`처럼 번호와 짧은 제목을 사용한다.
- 상태는 `Proposed`, `Accepted`, `Deprecated`, `Superseded` 중 하나를 사용한다.
- 한 ADR에는 하나의 결정만 담는다.
- 결정뿐 아니라 고려한 대안과 포기한 비용도 함께 적는다.
- 결정이 바뀌면 기존 ADR을 지우지 않고 새 ADR에서 대체한다.
