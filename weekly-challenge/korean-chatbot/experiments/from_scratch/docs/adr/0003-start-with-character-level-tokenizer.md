# ADR 0003. 첫 토크나이저는 character-level로 시작한다

- 상태: Accepted
- 날짜: 2026-06-18
- 관련 단계: `src/step01_tokenizer.py`

## 배경

한국어 문장을 모델에 넣으려면 먼저 문자열을 토큰으로 나누고, 각 토큰을
숫자 id로 바꿔야 한다. 한국어는 띄어쓰기와 형태소 문제가 있어 토큰화 방식에
따라 난도가 크게 달라진다.

## 문제

처음부터 좋은 한국어 토큰화 방식을 고르려 하면 형태소 분석기, subword
tokenizer, vocabulary 학습 등 여러 주제가 동시에 등장한다. 그러면
from-scratch 언어 모델의 기본 흐름을 이해하기 전에 전처리 복잡도에 막힐 수
있다.

## 고려한 선택지

1. character-level 토큰화
2. 공백 또는 정규식 기반 토큰화
3. 형태소 분석기 기반 토큰화
4. subword tokenizer 학습

## 결정

첫 구현은 character-level 토큰화로 시작한다.

## 결정 이유

- 구현이 가장 단순해서 전체 파이프라인을 빨리 연결할 수 있다.
- 한국어 형태소 분석 품질이나 외부 도구 설치 문제에 막히지 않는다.
- 텍스트, 토큰, id, 텐서, 모델 입력의 관계를 눈으로 확인하기 쉽다.
- 생성 결과가 부족하더라도 "왜 부족한지"를 다음 개선 주제로 삼기 좋다.

## 감수하는 단점

- 단어 의미 단위를 직접 반영하기 어렵다.
- 문장이 길어질수록 sequence length가 길어진다.
- 생성 결과가 글자 단위로 어색하게 나올 수 있다.
- 실제 챗봇 성능을 목표로 할 때는 더 나은 토큰화 방식이 필요할 수 있다.

## 나중에 다시 볼 조건

토큰 id, dataset, 모델 학습, auto-regressive 생성 흐름을 설명할 수 있게 되면
정규식 기반 토큰화, 형태소 분석기, subword tokenizer 중 하나를 다음 개선
후보로 검토한다.

## 결과 기록

`step01_tokenizer.py`에서 character-level encode/decode를 먼저 구현하고,
생성 품질의 한계를 이후 실험 기록과 다음 ADR에서 평가한다.
