# Week 04 - ML/DL 기초 실습

## 개요

이번 위클리 챌린지는 가상 데이터셋을 직접 만들고, 머신러닝과 딥러닝 모델을 단계적으로 학습해보는 실습이다.

처음에는 모든 과제를 하나의 이미지 데이터셋으로 이어가려고 했지만, 진행하면서 전통적인 머신러닝 모델을 이해하기에는 수치형 데이터가 더 적합하다고 판단했다. 그래서 1~4번은 2차원 수치형 이진 분류 데이터로 진행하고, 6번은 비선형 데이터셋, 7번은 별도의 합성 이미지 데이터셋으로 분리했다.

전체 목표는 단순히 정확도를 높이는 것이 아니라, 데이터 생성부터 분할, 전처리, 모델 학습, 평가, 시각화까지의 흐름을 직접 이해하는 것이었다.

## 회고

이번 과제에서 가장 어려웠던 부분은 모델 코드가 아니라 데이터 생성이었다. sklearn이나 PyTorch는 모델을 만드는 도구를 잘 제공하지만, 어떤 데이터를 만들고 그 데이터가 어떤 문제를 표현하는지 결정하는 일은 생각보다 훨씬 어려웠다.

처음에는 수치형 데이터가 아닌 이미지 데이터셋 하나로 1번부터 7번까지 한번에 깔끔하게 연결하려고 했다. 하지만 하나씩 알아가며 이미지 데이터셋을 생성하는거부터 꽤나 어려웠다. 추가로 K-NN, SVM, Naive Bayes 같은 전통적인 머신러닝 모델을 이미지 데이터로 바로 다루면 차원이 커지고, 모델의 동작 원리를 이해하기 전에 데이터 생성 자체가 너무 복잡해질 것 같았다. 그래서 중간에 방향을 바꿔 1~4번은 수치형 데이터로 단순화 하기로 결정했다.

특히 `StandardScaler`를 사용할 때 학습 데이터에는 `fit_transform()`을 적용하고, 검증/평가 데이터에는 `transform()`만 적용해야 한다는 점이 중요했다. 처음에는 예제 코드에 검증 데이터가 없어서 transform을 하지 않는 것처럼 이해했지만, 실제 이유는 데이터 누수를 막기 위해 학습 데이터의 기준만 사용해야 하기 때문이라는 것을 알게 되었다.

Naive Bayes를 사용할 때도 헷갈리는 지점이 있었다. 문제는 이진 분류라서 범주형 문제처럼 보였지만, 모델 선택 기준은 정답 라벨 `y`가 아니라 입력 데이터 `X`의 타입이었다. 이번 데이터의 `X`는 연속형 수치 데이터였기 때문에 `GaussianNB`를 사용하는 것이 적절했다. 이 과정에서 "분류 문제"와 "입력 데이터의 타입"은 따로 구분해서 봐야 한다는 점을 배웠다.

이번 주차 수업 내용과는 조금 무관할 수 있지만, 다른 사람의 코드를 보며 배운 점도 있었다. 처음에는 Perceptron, SVM, Random Forest, Naive Bayes를 각각 하나씩 모델 생성, 학습, 예측, 출력 코드로 모두 작성하고 있었다. 그러다 우연히 다른 사람이 `models = {}` 형태의 딕셔너리로 여러 모델을 묶고, 반복문으로 학습과 평가를 처리한 코드를 보게 되었다. 코드가 난잡하지 않고 훨씬 깔끔하게 정리되는 것을 보고 "와, 이렇게 작성할 생각을 못하고 있었다니"라는 생각이 들었다. 그래서 나도 같은 방식으로 코드를 바꿔보았다.

사실 아직도 다른 사람의 코드를 자세히 찾아보거나, 그 코드에 대해 피드백을 주고받는 것에는 익숙하지 않다. 혼자 작성하는 것에만 집중하다 보면 더 나은 구조를 볼 기회를 놓칠 수 있다는 생각이 들었다. 앞으로는 다른 사람이 같은 문제를 어떻게 풀었는지 더 자주 살펴보고, 그 안에서 배울 수 있는 상황을 더 많이 캐치하는 것에 익숙해지게 하고 싶다.

Data Augmentation도 단순히 데이터를 늘리면 더 많은 학습과 검증, 테스트 데이터가 생기기 때문에 왠만하면 성능이 좋아질거라고만 생각했다. 하지만 실제로는 원본 데이터가 충분히 쉽거나 많으면 증강 효과가 거의 보이지 않을 수 있었다. 그래서 학습 데이터 일부만 사용해 데이터가 부족한 상황을 만들고, Gaussian noise를 추가해 증강 전후를 비교했다. 한 포스팅 글에서도 증강은 항상 성능을 올리는 마법이 아니라, 데이터가 부족하거나 다양성이 부족한 상황에서 일반화를 돕는 방법이라는 글귀를 읽고 공감했다.

7번 CNN은 가장 AI의 도움을 많이 받은 부분이다. 처음 작성한 예제는 랜덤 이미지와 랜덤 라벨을 사용했기 때문에 모델이 학습할 수 있는 규칙이 없었다. 그래서 loss와 accuracy가 이상하게 움직였고, 결과를 해석하기도 어려웠다. 이후 원, 사각형, 삼각형 이미지를 직접 생성하고 CNN으로 분류하는 구조로 다시 작성했다. 이 부분은 혼자 완전히 설계했다기보다는 AI의 도움을 받아 구조를 정리했다. 다만 도움을 받는 과정에서도 왜 랜덤 라벨이 문제인지, CNN이 어떤 형태의 이미지 패턴을 학습해야 하는지, train/validation/test를 어떻게 나누어야 하는지를 확인하며 이해하려고 했다.

이번 과제를 하면서 저번주에 시각화를 했지만, 아직도 모르는 부분이 많고 뭔가 하나 새로운 그래프를 쓰거나 새로운 노이즈를 찍거나? 하려면 항상 함수를 찾아봐야 하는게 너무 번거롭게 느껴지기도 했다. 하지만 매번 모든걸 다 기억하고 외우고 할 수는 없다는 알렉스의 말씀에 내면의 스스로에 대한 화(?)를 누그러뜨릴 수 있었던 것 같다.

아직 CNN을 AI 없이 코드로 구현하기에는 실력이 부족한 것 같다. 그래도 이번 실습을 통해 데이터 생성, 모델 학습, 검증/평가, 시각화가 하나의 흐름으로 연결된다는 감각은 얻은 것 같다. 실제 이미지 데이터셋을 사용해서 한번 더 CNN까지 잘 수행해보고 싶은데, 시간적 여유가 많이 없을 것 같아 할 수 있을지 모르겠지만, 흥미롭기도, 여전히 순수 코딩 실력은 부족하구나 라는 사실을 인지하게 되는 4주차였던 것 같다.
