# WEEK03 - 데이터 분석 미니퀘스트

## 사용 데이터

### < 2-1, 2-2 > 서울시 공공자전거 따릉이 대여이력

- 출처: [서울 열린데이터광장 - 서울시 공공자전거 대여이력 정보](https://data.seoul.go.kr/dataList/OA-15182/S/1/datasetView.do)
- 사용 파일: `서울특별시 공공자전거 대여소별 이용정보(월별)_25.7-12.csv`
- 주요 데이터: 대여 일시, 대여소, 반납 일시, 반납 대여소, 이용 시간, 이용 거리 등
- 사용 목적: 실제 이용 기록을 바탕으로 지역별 따릉이 수요를 파악하기 위해 사용

따릉이 대여이력 데이터를 `DataFrame`으로 불러온 뒤, 대여소 정보를 기준으로 자치구별 대여건수를 집계했습니다.  
`groupby()`를 활용하여 자치구별 이용량을 비교하고, 자치구별 대여건수 합계가 100만 건 이상인 지역만 필터링했습니다.

단순히 조건에 맞는 데이터를 추출하는 데 그치지 않고, 대여량이 많은 지역에는 따릉이를 더 많이 배치할 수 있다는 실무적 활용 방향을 생각해 보았습니다.

### < 3-1 > 기상청 ASOS 월별 기온 데이터

- 출처: [기상청 기상자료개방포털 - 종관기상관측 ASOS](https://data.kma.go.kr/data/grnd/selectAsosRltmList.do?pgmNo=36)
- 사용 파일: `2025 청주 월별 기온 데이터.csv`
- 주요 데이터: 월별 평균기온, 평균 최고기온, 평균 최저기온, 강수량, 풍속 등
- 사용 목적: 시간의 흐름에 따른 월별 기온 변화를 시계열 그래프로 확인하기 위해 사용

2025년 1월부터 12월까지의 청주시 평균 최고기온과 평균 최저기온을 선 그래프로 시각화했습니다.  
두 선 사이의 범위를 색상으로 채워 계절에 따른 기온 변화와 월별 기온 범위를 쉽게 확인할 수 있도록 구성했습니다.

가로축에는 원본 데이터의 `일시` 컬럼에서 월을 추출하여 `01월`, `02월`과 같은 문자열 라벨을 사용했습니다.  
날짜 형식의 가로축에서 일부 월의 라벨이 자동으로 생략되는 현상을 해결하면서, 원본 컬럼을 활용하는 방법을 익혔습니다.

## 추가적으로 배운 점

- 기상청 CSV 파일은 `UTF-8`이 아니라 `CP949` 인코딩으로 제공될 수 있다는 점을 알게 되었습니다.
- `df.columns.str.strip()`을 사용하면 컬럼명 앞뒤의 불필요한 공백을 제거할 수 있습니다.
- 시계열 데이터는 반드시 일별 데이터일 필요가 없으며, 월별처럼 일정한 시간 간격으로 기록된 데이터도 시계열 데이터로 활용할 수 있습니다.
- Matplotlib은 날짜가 겹칠 가능성이 있으면 가로축 라벨 일부를 자동으로 생략할 수 있습니다.
- `plt.fill_between()`을 사용하면 두 선 사이의 범위를 색상으로 표현할 수 있어 그래프를 더 쉽게 해석할 수 있습니다.
- Google Colab에서 한글이 깨지는 현상이 발생하여, AI의 도움을 받아 나눔고딕 폰트를 설치하고 Matplotlib의 기본 폰트로 설정하는 방법을 AI의 도움을 받아 적용했습니다.
- Google Drive 공유 URL을 이용하면 제출된 노트북에서도 데이터를 다시 불러올 수 있어 열람과 재실행이 편리합니다.

## 회고

이번 과제에서는 서울시 따릉이 대여이력과 기상청 월별 기온처럼 실제로 활용할 수 있는 공공데이터를 다뤄 보았습니다.

따릉이 데이터에서는 `groupby()`와 조건 필터링을 문법 연습 + 빨리 끝내기 위한 코딩으로만 그치지 않고, 지역별 수요를 파악하여 자전거 배치 계획에 활용할 수 있다는 실무적 관점(?)으로 접근해 보고자 노력했습니다.

기상청 데이터 시각화에서는 처음에 주간 분석을 생각했지만, 실제 파일이 월별 데이터라는 점을 확인한 뒤 데이터 구조에 맞게 월별 시계열 그래프로 방향을 조정했습니다.

CSV 인코딩, Google Drive 공유 링크, Colab 한글 폰트, 날짜 라벨 표시처럼 실제 데이터를 사용할 때 생기는 작은 문제들과 직접 수행하며 겪는 한글 깨짐 문제, 컬럼명을 제 입맛대로 만드는데 마주한 문제를 함께 해결해 볼 수 있었습니다.

상대적으로 복잡도가 낮은 데이터를 추천받아 진행하며 어떤 의도를 담아 그룹화하고 시각화할지 고민하는 과정이 꽤나 흥미롭고 재밌었습니다.

앞으로도 코드를 작성하기 전에 데이터의 단위와 컬럼을 먼저 확인하고, 분석 결과를 어디에 활용할 수 있을지 생각하며 더 좋은 결과를 낼 수 있으면 좋겠습니다.
