## [핵심 배운 내용]

- 전이 학습은 사전 학습된 가중치를 활용해 기본 시각 특징을 빠르게 잡고, 일반화 성능을 높이는 데 도움을 준다.
- 과적합은 학습 데이터에 지나치게 맞춰져 실제 데이터에서 오차가 커지는 현상이며, 정규화·드롭아웃·조기 종료 등으로 완화할 수 있다.
- NLP에서는 텍스트를 모델이 계산 가능한 형태로 바꾸기 위해 토큰화, 형태소 분석, 원핫 인코딩, 워드 임베딩 등을 사용한다.
- RNN 계열 모델은 순차 데이터 처리에 강점이 있고, LSTM은 RNN의 장기 기억 한계를 보완한다.

### 전이 학습

#### 사전 학습 가중치(Pre-trained Weights)를 사용하는 이유

- 이미지 데이터는 서로 다르게 보이지만, 저차원에서 보면 결국 **선, 면, 경계(Edge), 음영, 질감** 같은 공통적인 시각 요소로 구성된다.
- ImageNet 같은 거대 데이터셋으로 학습된 사전 가중치는 도메인과 관계없이 이러한 **기본 시각 특징을 포착하는 능력**을 이미 갖고 있다.
- 따라서 모델을 백지 상태에서 학습시키는 대신, 검증된 시각 템플릿을 기반으로 시작할 수 있다.
- 이는 일반화 성능 향상과 관련이 있으며, 과적합을 줄이는 데도 도움을 줄 수 있다.

#### 사전 권장 가중치를 쓰는 이유

- `weight=None`으로 완전히 새로 학습하는 것보다, 권장되는 사전 학습 가중치를 사용하는 편이 일반적으로 도움이 된다.
- 데이터와 상황마다 차이는 있지만, 권장 가중치가 존재하는 이유는 실제로 성능과 일반화 측면에서 유리한 경우가 많기 때문이다.

#### 전이 학습의 종류

- **파인튜닝(Fine-tuning)**: 사전 훈련된 모델을 현재 데이터에 맞게 미세 조정하여 성능을 최적화한다.
- **특징 추출(Feature Extraction)**: 사전 훈련된 모델을 특징 추출기로 활용한다.
- **제로샷 러닝(Zero-shot Learning)**: 별도의 예시 없이 모델이 가진 기존 지식과 추론 능력만으로 처음 보는 문제를 해결한다.
  - 의미 공간(Semantic Space)을 활용한다.

### 과적합(Overfitting)

- 학습 데이터를 과하게 학습해, 실제 데이터에 대해서는 오차가 증가하는 현상이다.

#### 방지 방법

- **교차 검증**: 데이터를 여러 세트로 나누어 모델을 평가한다.
- **정규화**: 모델의 복잡성을 줄여 과적합을 방지한다.
- **추가 데이터 확보**: 학습 데이터를 늘려 모델의 일반화 능력을 높인다.
- **드롭아웃**: 일부 뉴런을 무작위로 학습에서 제외해 과적합을 줄인다.
- **조기 종료**: 검증 데이터의 오차가 증가하기 시작하면 학습을 멈춘다.

### 하이퍼파라미터 튜닝

모델 학습 흐름은 다음과 같이 이해할 수 있다.

1. 피드 포워드
2. 손실 함수 값 계산
3. 역전파로 그래디언트 계산
4. 최적화를 통해 매개변수 업데이트

#### 정규화 파라미터

- 손실의 총합은 다음과 같이 표현할 수 있다.

```text
Loss_total = Loss + λ(정규화 파라미터) * Penalty
```

- 정규화 파라미터는 과적합을 방지하기 위해 가중치에 패널티를 부여하는 값이다.
- 모델이 지나치게 커지고 복잡해지는 것을 막는 역할을 한다.
- 대표적인 규제로는 **L1, L2 규제**가 있으며, 각각 라쏘(Lasso), 릿지(Ridge) 규제와 연결된다.
- 일반적으로 `p-norm`을 사용한다.

$$
(x_1^p + x_2^p)^{1/p}
$$

## NLP (Natural Language Processing)

### 언어 모델

- 앞의 단어가 주어졌을 때, 다음 단어가 나타날 확률을 빈도 기반으로 계산하는 모델이다.

### N-gram Language Model

- `n-1`개의 단어를 기준으로 다음 단어의 조건부 확률을 계산하는 전통적인 통계 기반 언어 모델이다.
- 한국어는 주로 형태소 기반으로 접근한다.

### 형태소 분석

- 문장을 의미를 가진 가장 작은 단위인 형태소로 나누는 전처리 과정이다.
- 직접 구현할 수도 있지만, 일반적으로는 기존 형태소 분석기를 활용한다.

### 원핫 인코딩(One-hot Encoding)

- 범주형 값을 **해당 범주 위치만 1이고 나머지는 0인 고정 길이 벡터**로 표현하는 방식이다.
- 정수 인코딩된 토큰 ID를 인덱스로 사용해, 어휘 크기만큼의 벡터에서 해당 인덱스만 1로 켠다.
- 범주형 값을 순서나 크기 의미 없이 모델이 계산할 수 있는 숫자 벡터로 표현하기 위해 사용한다.

#### 핵심 특징

- 각 토큰이 서로 독립적으로 표현된다.
- 순서나 크기 의미가 없다.
- 값이 어떤 토큰인지는 벡터 내 위치로 명확히 드러난다.
- 어휘 크기만큼 벡터가 커지기 때문에, 어휘가 많아질수록 메모리 낭비가 커진다.
- 이러한 표현을 **희소 표현(Sparse Representation)**이라고 한다.

#### 한계

- **차원의 저주**: 단어가 10만 개라면 단어 하나를 표현하기 위해 숫자 10만 개짜리 벡터가 필요하다.
- 대부분의 값이 0인 희소 표현이므로 메모리 효율이 낮다.
- 단어 간 의미적 관계를 표현하지 못하는 **의미 상실** 문제가 있다.
- 워드 임베딩은 이러한 한계를 해결하기 위한 표현 방식이다.

### 워드 임베딩(Word Embedding)

- 단어를 고정된 크기의 실수 벡터로 변환해, 단어 간 의미적 유사성과 사용 맥락의 유사성을 반영하는 표현 방식이다.
- 원핫 인코딩보다 낮은 차원의 벡터로 단어를 표현한다.

#### 원핫 인코딩과의 차이

- 원핫 인코딩은 각 단어를 독립적인 축으로 표현한다.
- 워드 임베딩은 단어들 간의 의미적 거리와 방향을 벡터 공간에서 표현할 수 있다.
- 따라서 단어 간 유사성을 더 잘 찾을 수 있다.
- 단어 간 유사성을 벡터 공간에서 직접 계산할 수 있다.

### RNN (Recurrent Neural Network)

- 이전 시점의 hidden state를 다음 시점 계산에 다시 사용하며 순차 데이터를 처리하는 **순환 구조의 인공 신경망**이다.
- 단계나 시간 흐름이 중요한 문제에서는 RNN 계열 모델을 우선적으로 고려할 수 있다.
- RNN이 이전 정보를 반영할 수 있는 이유는, 이전 시점에서 계산된 hidden state가 다음 시점의 입력 계산에 함께 사용되기 때문이다.
- 이러한 구조를 RNN의 순환(Recurrent) 연결이라고 한다.

### BPTT (Backpropagation Through Time)

- RNN은 층이 여러 개인 것이 아니라, 하나의 층이 시간 방향으로 반복해서 사용되는 구조다.
- 따라서 역전파도 시간 방향으로 펼쳐서 수행한다.
- 이를 **BPTT(Backpropagation Through Time)**라고 한다.

### LSTM (Long Short-Term Memory)

- RNN은 문장이 길어질수록 앞쪽에서 배운 중요한 정보를 뒤로 갈수록 잊어버리는 한계가 있다.
- LSTM은 중요한 기억을 별도의 전용 통로에 담아두어, 긴 문장에서도 필요한 정보를 오래 유지할 수 있도록 제어하는 신경망이다.
- 직관적으로는 **RNN 구조에 cell state를 추가해, 정보가 시간에 따라 쉽게 사라지지 않도록 만든 모델**로 이해할 수 있다.

#### Cell State의 역할

- 기존 RNN은 과거 정보가 다음 시점으로 넘어갈 때 활성화 함수(`tanh`)를 거치며 계속 왜곡되거나 약해질 수 있다.
- LSTM의 cell state는 정보가 비교적 일직선으로 흐르는 통로 역할을 한다.
- 덕분에 정보가 변질되거나 사라지는 것을 줄이고, 더 먼 시점까지 전달할 수 있다.
