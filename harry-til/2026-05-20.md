## [핵심 배운 내용]

#### 데이터베이스

- 사용하는 이유 ? ⇒ 데이터를 구조화해서 저장하여, 데이터 안정성과 신뢰성 보장하기 위해서
- 어떻게 자료를 구조화할 것인가 ? 이게 주된 고민


#### 정규화

- 관계형 데이터베이스의 설계에서 중복을 최소화하게 데이터를 구조화하는 프로세스
- 사용하는 이유 ? ⇒ 무결성을 유지하고 저장 용량을 줄이기 위해서
- **이상현상 ? ⇒ 데이터베이스 설계가 잘못되어서 발생하는 문제점**
    - 삽입 이상 - 삽입 시, 원하지 않는 정보까지 입력되는 현상 발생
    - 삭제 이상 - 삭제 시, 원하지 않는 정보까지 삭제되는 데이터 손실 현상 발생
    - 갱신 이상 - 갱신 시, 데이터 중복으로 인해 일관성이 없어지는 현상 발생
    - 한번 일어나면 너무 막대한 피해를 입힐 수 있기에, 자주 있지 않는 일이라도 엄청 대비해야 한다.


#### ERD

- 처음 본다면 꼭 보기.
- ddl로 어떻게 연결될지
- 개체와 관계를 한 눈에 알아볼 수 있도록 그려놓는 도표
- 사용 이유 ? ⇒ 데이터베이스의 구조를 시각적으로 표현하여 데이터 간의 관계와 구조를 명확하게 이해하기 위해서
- DB → RDB → SQL → ERD

#### FastAPI

- 함수 선언 def VS async
- 마이크로 프레임워크
- 성능을 100프로 쓰려면 데이터베이스 드라이버나 외부 통신 라이브러리도 비동기(`asyncpg`, `httpx` 등)를 써야함
- Starlette(비동기 웹 서버)
- **Uvicorn은 진짜 '서버(하드웨어와 소통하는 문지기)'이고, Starlette은 '웹 프레임워크(요청을 분류하는 안내원)’**
    
    **[ 가장 바깥쪽 (인터넷 망) ]**
    
    ⬇️ (1) HTTP 요청 들어옴
    🧱
    
    **Uvicorn** (웹 서버 / ASGI Server) : 요청을 받아서 파이썬이 읽을 수 있게 ASGI 데이터로 번역해 줌.
    ⬇️ (2) 번역된 데이터 전달
    🛣️
    
    **Starlette** (웹 툴킷 / ASGI Framework) : 라우팅(함수 실행 주소 찾기), 쿠키/세션 관리, 웹소켓 연결 등 기본 뼈대 처리.
    ⬇️ (3) 정리된 데이터 전달
    💎
    
    **FastAPI** (API 프레임워크) : Pydantic으로 데이터 검증하고, Swagger 문서 자동으로 그려줌.
