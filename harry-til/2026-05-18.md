# 📝 TIL (Today I Learned)

## 📌 핵심 배운 내용
* **파이썬 가변 인자 처리 (`*args`, `**kwargs`)**
  * `*args`: 함수 호출 시 넘겨준 여러 개의 일반 인자들을 함수 내부에서 **튜플(Tuple)**로 묶어서 받는다.
  * `**kwargs`: 함수 호출 시 넘겨준 키워드 인자들(`key=value`)을 함수 내부에서 **딕셔너리(Dictionary)**로 묶어서 받는다. 정의되지 않은 방식으로 인자를 넘기면 `TypeError`가 발생한다.

<summary>🌐 HTTP (HyperText Transfer Protocol)</summary>

* **개념:** HTML, CSS, JS, 이미지 등 일반적인 텍스트를 넘어선 구조화된 하이퍼텍스트 문서를 전송하기 위해 사용되는 웹 표준 통신 규약
* **사용 이유:** 웹 서버와 클라이언트가 서로 오해 없이 데이터를 주고받을 수 있도록 규격화된 약속을 정의하기 위함
* **HTTP Message:** 통신 시 주고받는 실제 편지 양식
  * `HTTP Request Message` ➔ 클라이언트가 서버에 보내는 요청 편지
  * `HTTP Response Message` ➔ 서버가 클라이언트에 돌려주는 응답 편지
* **구조:** `Header`(메타 정보, 데이터 길이 등)와 `Body`(본문 데이터)로 구성
  * ⚠️ **`\r\n\r\n` (엔터 두 번) 코드로 헤더와 바디의 경계선을 구분함**
* **대표적인 클라이언트:** 웹 브라우저
* **HTTP 요청 메소드 (용건 표시):**
  * `GET`: 서버로부터 리소스 조회 요청 (Body 포함 X)
  * `POST`: 서버에 데이터를 전송하여 새로운 리소스 생성/업데이트 (Body 포함 O)
  * `PUT`: 목적지 주소에 있는 리소스 **전체**를 대체/업데이트
  * `PATCH`: 목적지 주소에 있는 리소스를 **부분적으로** 수정
  * `DELETE`: 서버의 특정 리소스를 삭제 요청
  * *참고 (RPC):* Remote Procedure Call (원격 함수 호출 방식)
* **HTTP 상태 코드 (결과 도장):**
  * 요청의 처리 결과를 3자리 숫자로 명확하게 나타내는 통신 규칙
  * `1xx` (정보), `2xx` (성공 - e.g., 200 OK), `3xx` (리다이렉션), `4xx` (클라이언트 오류 - e.g., 400 Bad Request), `5xx` (서버 오류 - e.g., 500 Internal Server Error)
* **HTTP URL (리소스 주소):**
  * 웹 상의 자원을 식별하기 위한 경로 명세
  * 구조: `Scheme://Domain:Port(생략가능)/resource?parameters`
  * 포트(Port)까지가 해당 서버를 찾는 주소이며, 이후 경로는 서버 내부의 자원 위치를 가리킴
</details>

<details>
<summary>📦 JSON (JavaScript Object Notation)</summary>

* **개념:** 시스템(프로그램) 간에 데이터를 가볍고 구조화된 형태로 주고받기 위한 공유 텍스트 포맷
* **특징:** 기본적으로 Key-Value 형태의 딕셔너리 구조(`{}`) 및 배열 구조(`[]`)를 지원함
* **FastAPI와의 연동:** FastAPI에서는 파이썬의 `dict`나 `list`를 반환하기만 하면 자동으로 JSON 포맷으로 변환해 줌
  * **직렬화 (Serialization):** 프로그램 메모리 내부의 데이터 객체를 전송/저장 가능한 텍스트(JSON 문자열) 형태로 변환하는 과정
  * **역직렬화 (Deserialization):** 수신한 JSON 텍스트를 프로그램이 이해할 수 있는 메모리 객체로 복원하는 과정
* *💡 실무 한줄평:* JSON과 REST는 백엔드 개발자로서 앞으로 평생 붙어 다닐 핵심 짝꿍이다.
</details>

<details>
<summary>📐 REST (Representational State Transfer)</summary>

* **개념:** HTTP의 기존 인프라를 그대로 활용하여, **자원(Resource)을 명사형 URL로 식별**하고 **행동은 HTTP Method로 정의**하여 자원의 상태를 JSON 등으로 교환하는 소프트웨어 아키텍처 스타일
* **API (Application Programming Interface):** 서로 다른 응용 프로그램들이 대화할 수 있도록 규칙을 정해둔 소통 창구(인터페이스)
</details>

<details>
<summary>🔌 네트워크 & 노드 기본 개념</summary>

* **네트워크:** 기기들 간의 연결을 통해 자원을 공유하는 시스템 (범위에 따라 LAN, MAN, WAN으로 분류)
* **노드 (Node):** 네트워크 내에서 데이터를 생성, 처리, 송수신하는 개별 장치/지점 (서버, 라우터, 스위치 등)
  * 각 노드는 MAC 주소 또는 IP 주소와 같은 고유한 식별 주소를 가지며 데이터 흐름을 형성함
* **TCP/IP:** 인터넷 통신을 가능하게 하는 표준 프로토콜 스택(규칙의 집합)
* **소켓 (Socket):** 개발자가 네트워크 통신 프로그래밍을 쉽게 할 수 있도록 운영체제가 추상화하여 제공하는 연결 창구
* **Localhost:** 네트워크 상에서 '자기 자신(Self)'의 컴퓨터를 가리키는 루프백 주소 (`127.0.0.1`)
</details>

<details>
<summary>⚡ FastAPI & Web Server Architecture</summary>

* **WSGI vs ASGI:**
  * `WSGI`: 웹 서버와 파이썬 웹 애플리케이션 간 통신을 정의한 동기식 표준 규격 (구현체: Gunicorn)
  * `ASGI`: WSGI의 한계를 넘어 **비동기(Asynchronous)** 통신까지 지원하는 차세대 표준 규격 (FastAPI가 채택)
* **FastAPI 특징:** 비동기 기반으로 API를 빠르고 생산성 높게 구축할 수 있는 현대적인 웹 프레임워크
  * `@app.get("/")`: 지정한 URL 경로로 GET 요청이 들어왔을 때 바로 아래에 정의된 파이썬 함수를 매핑하여 실행해 주는 데코레이터
  * `fastapi[standard]`: uv 환경에서는 `uv add fastapi --extra standard` 혹은 `uv add fastapi` 방식으로 의존성을 손쉽게 관리 가능
  * 🔒 **Pydantic:** FastAPI에서 데이터 검증 및 타입 체크를 위해 사용하는 핵심 라이브러리 (매우 중요, 추후 심화 학습 필요)
</details>

<details>
<summary>🧱 프레임워크 vs 라이브러리</summary>

* **프레임워크 (Framework):** 애플리케이션의 전체적인 골조(흐름의 제어권)를 제공하며, 개발자는 그 규칙 안에서 코드를 작성함 (제어의 역전 현상 발생)
* **라이브러리 (Library):** 개발자가 주도권을 가지고 필요할 때마다 부품처럼 호출해서 사용하는 코드의 집합
* *Reinvent the wheel (바퀴의 재발명):* 이미 잘 만들어진 라이브러리가 있는데 이를 인지하지 못하고 직접 바닥부터 짜고 있는 현상
  * *생각해볼 점:* 생산성 측면에서는 비효율적일 수 있으나, 개념을 깊이 이해하고 커스텀 기능이 필요한 공부/도전 단계에서는 무조건 나쁜 행위는 아니다. 생산성과 다양성 간의 밸런스가 중요함.
</details>

## ❓ 수업 중 생긴 궁금증 & 답변 정리

### 1. 소켓(Socket)과 포트(Port)의 차이
* **포트 (Port):** 외부에서 특정 프로세스(프로그램)를 찾아오기 위한 통신 통로 번호 (**건물의 방 번호**)
  * 고정되거나 표준화된 숫자로 매핑되는 경우가 많음 (HTTP: 80, SSH: 22)
* **소켓 (Socket):** 프로세스가 네트워크를 통해 실제 데이터를 주고받기 위해 동적으로 개방하는 창구 (**방 안에 놓인 전화기 단자**)
  * 통신이 필요할 때마다 소프트웨어적으로 생성 및 소멸됨

### 2. 패키지 매니저 삼총사 비교: Homebrew vs uv vs pip
* **🛠️ Homebrew:** 컴퓨터 OS 전체를 아우르며 인프라 프로그램(`Git`, `Nginx`, `Database` 등)을 깔아주는 **이삿짐센터**
* **⚡ uv:** Rust 기반으로 제작되어 파이썬 가상환경 빌드 및 라이브러리 추가를 믿을 수 없을 만큼 빠르게 처리하는 **차세대 파이썬 총괄 매니저**
* **📦 pip:** 파이썬 순정 패키지 매니저 시스템이지만, 속도가 느리고 가상환경 분리 기능이 약해 **최근 트렌드에서는 `uv`로 대체되는 추세**

💡 **실무 협업 흐름 (Workflow):**
1. `Homebrew`로 컴퓨터에 `Git`, `DB` 등 공통 인프라를 세팅한다.
2. 프로젝트 폴더 내에서 `uv`로 격리된 파이썬 가상환경을 생성한다.
3. `uv pip install fastapi uvicorn` 명령어로 필요한 라이브러리를 초고속으로 다운받아 개발을 시작한다.
