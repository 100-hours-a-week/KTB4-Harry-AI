- Transformer에서는 토큰 임베딩, 포지셔널 임베딩, Q/K/V 가중치, FFN, Linear layer 등이 학습 대상이 된다.

## Transformer 개요

### Transformer

- Transformer는 **Attention을 중심으로 시퀀스 전체의 문맥을 계산하는 신경망 구조**다.
- Encoder와 Decoder로 구성된 Sequence-to-Sequence 모델로 볼 수 있다.
- RNN처럼 순차적으로만 처리하지 않고, 시퀀스 전체를 병렬적으로 계산할 수 있다는 장점이 있다.
- FCNN, CNN, RNN은 기본 개념과 아이디어를 잘 이해해두고, Transformer는 현재 State-of-the-Art 모델의 핵심 구조이므로 더 깊게 이해해야 한다.

### Sequence

- Sequence는 **순서가 의미를 가지는 요소들의 나열**이다.
- 단순한 집합이 아니라, 순서가 바뀌면 의미도 달라지는 데이터 구조다.
- 자연어 문장은 단어의 순서에 따라 의미가 달라지므로 대표적인 sequence 데이터다.

## Transformer 구조

### 전체 흐름

```text
[입력 토큰 시퀀스]
  ↓ 토큰 임베딩 + 포지셔널 인코딩
[인코더 입력]
  ↓ 인코더 N층
    (Self-Attention + FFN + Residual Connection + LayerNorm)
[인코더 출력: 문맥 표현]

[디코더 입력: 이전 출력 토큰]
  ↓ 토큰 임베딩 + 포지셔널 인코딩
  ↓ 디코더 N층
    (Masked Self-Attention → Cross-Attention → FFN
     + Residual Connection + LayerNorm)
[디코더 출력]
  ↓ Linear Layer (d_model → vocab_size)
  ↓ Softmax
[다음 토큰 확률 분포]
```

### Encoder

- Encoder는 입력 문장의 **각 토큰을 문장 전체의 문맥이 반영된 벡터로 바꾸는** Transformer의 입력 처리 블록이다.
- 결과물은 각 토큰에 대한 context vector다.
- 예를 들어 `"나는 학교에 간다"`라는 문장이 있다면, Encoder는 각 토큰을 다음과 같은 문맥 벡터로 바꾼다.
  - `나는`의 문맥 벡터
  - `학교에`의 문맥 벡터
  - `간다`의 문맥 벡터

### Decoder

- Decoder는 Encoder의 출력과 이전까지 생성된 토큰을 참고해 다음 토큰을 순차적으로 생성하는 Transformer의 구성 요소다.
- 번역처럼 입력 문장을 다른 출력 문장으로 바꾸는 task에서는 Encoder-Decoder 구조를 함께 사용한다.
- 우리가 자주 접하는 LLM은 보통 **Decoder-only 모델**이다.
- Decoder-only 구조만으로도 텍스트 생성, 질의응답 등 다양한 작업을 수행할 수 있다.

### Teacher Forcing

- Teacher Forcing은 학습할 때 Decoder에 모델이 직접 만든 출력 대신 **정답 문장**을 입력으로 넣어주는 학습 방식이다.
- 학습 중에는 정답 문장을 한 칸 오른쪽으로 밀어 Decoder 입력으로 넣고, 원래 정답 문장은 모델이 맞혀야 할 목표값으로 사용한다.
- 이를 통해 학습 초기에 잘못된 예측이 계속 누적되는 문제를 줄일 수 있다.

### Add & Norm

- Add & Norm은 각 블록의 계산 결과를 안정적으로 다음 단계에 전달하고, 깊은 층에서도 학습이 잘 이어지도록 돕는 구조다.
- `Add`는 residual connection을 의미하고, `Norm`은 Layer Normalization을 의미한다.
- 깊은 Transformer 구조에서 정보와 기울기가 안정적으로 흐르도록 돕는다.

### FFN의 역할

- **토큰 간 관계는 Attention에서 계산하고, FFN은 각 토큰 위치의 벡터를 개별적으로 변환한다.**
- 즉 Attention은 문맥 관계를 반영하고, FFN은 각 위치의 표현을 더 풍부하게 바꾸는 역할을 한다.

### Linear Layer와 Softmax

- Decoder의 마지막 층을 통과한 출력은 아직 실제 단어가 아니라, 각 위치에 대한 벡터다.
- 모델은 이 벡터를 실제 토큰 후보 점수로 바꾸기 위해 Linear layer를 사용한다.
- 이후 Softmax를 거쳐 다음 토큰에 대한 확률 분포를 만든다.

## Input Pipeline

### 전체 흐름

```text
자연어 문장
  → 토큰화
  → 토큰 ID
  → 토큰 임베딩
  → 포지셔널 인코딩
  → Transformer 입력 벡터
```

- 자연어 문장은 바로 모델에 들어갈 수 없기 때문에 여러 전처리 단계를 거친다.
- 토큰화로 문장을 토큰 단위로 나누고, 각 토큰을 정수 ID로 바꾼다.
- 정수 ID는 임베딩을 통해 벡터로 변환된다.
- Transformer는 순환 구조가 없기 때문에 포지셔널 인코딩으로 위치 정보를 추가한다.

### Token

- Token은 모델이 구분하는 자연어의 기본 단위다.
- 의미를 가지는 최소 단위처럼 볼 수 있지만, 실제로는 tokenizer의 분절 규칙에 따라 단어, 서브워드, 문자 단위가 될 수 있다.

### Tokenizer

- Tokenizer는 입력 텍스트를 분절 규칙에 따라 token으로 나누고, 어휘 사전을 참조해 정수 ID 시퀀스로 변환하는 전처리 모듈이다.
- 필요하면 `BOS`, `EOS`, `PAD` 같은 특수 토큰도 추가한다.

#### Tokenizer를 만들 때 필요한 것

1. 텍스트를 어떤 단위로 나눌지 정하는 분절 규칙
2. 토큰과 정수 ID를 연결하는 어휘 사전

```text
Apple → 3
나는 → 1024
```

### Subword Tokenization

- Subword tokenization은 단어를 통계적 기준에 따라 더 작은 부분 문자열로 분리하는 토큰화 기법이다.
- 처음 보는 단어도 더 작은 단위의 조합으로 표현할 수 있어, 어휘 사전의 크기를 줄이고 OOV(Out-of-Vocabulary) 문제를 완화할 수 있다.

### Token Embedding

- Token embedding은 토큰 ID를 상대적으로 낮은 차원의 벡터 공간에 매핑하는 과정이다.
- 예를 들어 `나는 → 1024 → (0.1, 0.3, ..., 0.7)`처럼 정수 ID가 학습 가능한 벡터 표현으로 바뀐다.
- 이 벡터의 값은 모델 학습 과정에서 조정된다.

### Padding

- Padding은 길이가 서로 다른 시퀀스를 동일한 길이로 맞추기 위해, 짧은 시퀀스의 빈자리를 `pad_token_id`로 채우는 전처리 기법이다.
- 주로 batch 처리를 위해 길이를 맞추거나, 특정 위치를 attention 계산에서 제외하고 싶을 때 사용한다.

### Positional Encoding

- Positional encoding은 시퀀스 내 위치 정보를 벡터나 연산 형태로 반영해, 모델이 토큰의 순서를 구분할 수 있게 하는 기법이다.
- Transformer는 RNN처럼 순환 구조가 없기 때문에, 토큰의 순서를 자체적으로 알 수 없다.
- 따라서 위치 정보를 명시적으로 주입해야 한다.

#### 위치 정보를 넣는 방식

- 사인/코사인 함수로 위치별 벡터를 생성해 토큰 임베딩에 더한다.
- 모델에 따라 학습 가능한 positional embedding을 사용하기도 한다.
- 상대 위치 기반 방식을 사용해 토큰 간 상대적 거리 정보를 반영하기도 한다.

#### 이해 포인트

- RNN은 순차 구조를 통해 순서를 자연스럽게 반영한다.
- Transformer는 순환 구조가 없기 때문에 **순서 정보를 위치 정보로 바꿔 주입한다**고 이해할 수 있다.
- 즉, 순서를 절대 좌표나 상대 위치 정보로 표현해 순환 구조의 부재를 보완한다.

## Attention

### Attention

- Attention은 **현재 토큰이 참고해야 할 다른 토큰을 찾고, 관련성이 높은 정보에 더 큰 비중을 두어 새로운 표현을 만드는 연산 방식**이다.
- 단어들 사이에 어떤 관계가 있는지를 수치화한다.
- 문장 안에서 멀리 떨어져 있어도 의미상 중요한 토큰을 직접 참고할 수 있다.

### Query, Key, Value

- Attention은 `Query`, `Key`, `Value`라는 세 가지 벡터를 사용한다.
- 이때 Q, K, V를 만드는 가중치 행렬도 학습 대상이다.

#### Query

- Query는 현재 토큰이 참고해야 할 정보를 찾기 위한 기준이다.
- 현재 토큰이 “무엇을 찾고 싶은지”를 나타낸다.

#### Key

- Key는 각 토큰이 자신의 특징을 나타내기 위해 제공하는 비교 기준 벡터다.
- Query와의 관련성 점수를 계산하는 기준이 된다.

#### Value

- Value는 attention 가중치에 의해 가중합되어 최종 출력을 구성하는 내용 표현 벡터다.
- 어떤 토큰을 참고할지 계산하는 과정과, 실제로 가져올 정보를 분리하기 위해 사용한다.

### Scaled Dot-Product Attention

$$
Attention(Q, K, V) = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

- `QK^T`로 Query와 Key의 유사도를 계산한다.
- `softmax`를 통해 각 토큰을 얼마나 참고할지에 대한 가중치를 만든다.
- 마지막으로 이 가중치를 Value에 곱해 최종 attention 출력을 만든다.

#### 왜 \(\sqrt{d_k}\)로 나누는가?

- 내적 값이 너무 커지면 softmax 결과가 0이나 1에 가까운 극단적인 값이 될 수 있다.
- 이렇게 되면 gradient가 작아져 학습이 어려워질 수 있다.
- `\sqrt{d_k}`로 나누는 scaling을 통해 값을 적절한 범위로 유지한다.

## 모델별 학습 대상

모델을 공부할 때는 “어떤 값이 학습되는가?”를 함께 생각해보면 구조를 이해하는 데 도움이 된다.

| 모델 | 주요 학습 대상 |
| --- | --- |
| FCNN | 가중치, 편향 |
| CNN | 합성곱 필터(가중치), 편향 |
| NPLM | 단어 임베딩 가중치, FCNN 가중치 |
| RNN | 입력 가중치, hidden state 관련 가중치, 출력 가중치, 편향 |
| Transformer | 토큰 임베딩, 포지셔널 임베딩, Q/K/V 가중치 행렬, FFN 가중치, Linear layer 가중치 |
