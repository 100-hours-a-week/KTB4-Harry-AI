#### 미들웨어

- 요천이 라우터에 도달하기 전과, 응답이 클라이언트로 나가기 전의 전체 흐름을 감싸서 실행되는 코드
    - 애플리케이션 전체 요청 흐름에 공통으로 개입하는 장치 (= 중간에 있는 소프트웨어)
- 미들웨어를 사용하기 적합한 작업
    - **CORS 처리**
        - 브라우저 요청을 허용/차단하기 위해 모든 응답 헤더에 공통 적용해야 하므로 미들웨어가 필수
    - **전역 Rate Limiting**
        - 과도한 요청을 가능한 한 빨리 차단해 서버 자원을 보호해야 하므로, 라우터 이전 단계에서 처리하는 미들웨어가 적합
    - **요청 처리 시간 측정**
        - 요청 시작부터 응답 완료까지의 전체 시간을 측정하려면, 요청과 응답을 모두 감쌀 수 있는 미들웨어가 필요
- web app —— middleware(web app server (FastAPI, uvicorn)) —— OS
- request handler —— middleware —— ASGI server

#### HTML

- = HyperText Markup Language / 텍스트를 뛰어넘는 마크업 언어
- 사용하는 이유 : 웹 브라우저가 문서를 해석해 시각적으로 홈페이지 구조를 표현할 수 있게 하기 위해서
    - **<div>** - 구역을 나누거나 그룹화하기 위한 블록
    - **<h1> ~ <h5>** - 다른 크기의 제목을 정의, 문서 구조와 정보의 중요도를 나타냄
    - **<p>** - 문단을 정의하는 블록 요소, 텍스트의 단락을 구분하여 표시
    - **<br>** - 줄 바꿈을 나타내는 요소

#### Fetch

- 정의 :컴퓨터가 무엇을 해야 할지 알려주는 지시를 읽고, 그 지시에 따라 필요한 정보를 찾아 실행하는 첫 단계
- 사용 이유 : 지시된 작업을 정확히 수행하기 위해 필요한 정보를 불러오기 위해서
    - **`then()`** 메서드를 사용하여 응답을 처리
    - **`.json()`**을 사용하여 응답은 다양한 형태로 변환될 수 있음
    - **`catch()`** 메서드를 사용하여 오류를 처리
- 풀어주는 다른 오리진의 접속을 허용하겠다고 의사표시 하는 주체 → 서버 정책
